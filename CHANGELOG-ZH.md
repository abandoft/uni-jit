## 0.1.6

- 新增完整 QuickJS 数值计数循环编译，覆盖 Float64 局部状态、算术赋值、有序条件重置、SSA 合流、回边安全点、GC 托管原生调用，以及在已验证工作负载上超过 V8 Jitless 的位级精确目标基准。
- 新增完整 PocketPy `range(count)` 数值循环编译，覆盖缩进校验、Float64 局部状态、条件更新、SSA 合流、回边安全点、GC 托管原生调用，以及在已验证工作负载上超过 CPython 3.14.6 解释器和 JIT 模式的位级精确目标基准。
- 修复 CFG 编译元数据，确保包含安全点的函数始终通过安全调用 API 获得执行上下文。
- 修复多个目标参数共享物理寄存器时的 CFG 并行边复制，防止非局部循环状态在必需的栈发布前被覆盖。
- 新增有序 Float64 CFG 比较、IEEE-754 无序结果为 false 的语义，以及全部三种架构上的原生编码。
- 新增类型化 Float64 CFG 参数、常量、算术、循环携带边校验、解释器执行，以及 AArch64、x86-64 与 RISC-V 64 原生降级。

## 0.1.5

- 新增由单一源码驱动的 QuickJS、UniJIT、V8 Jitless 与 V8 语言内循环基准，包含固定 Node 执行环境、位级精确校验、引擎元数据与 CI 结果留存。
- 新增由单一源码驱动的 PocketPy、UniJIT、CPython 3.14.6 与 CPython 3.14.6 JIT 语言内循环基准，包含严格运行模式检查、官方包校验与 CI 结果留存。
- 将无 release 的 LuaJIT 参考依赖推进到上游 v2.1 分支最新提交，同时继续把所有参考构建生成状态限制在 `build/` 下。
- 消除可证明恒通过的 Float64 非零守卫，使 PocketPy 的已知非零常量除法继续使用无检查原生调用快路径。

## 0.1.4

- 为类型化 SSA 与全部原生后端新增 IEEE-754 Float64 除法，并接入受守卫保护的 Lua 5.5 和 QuickJS 前端。
- 新增覆盖加、减、乘、除的链式 Float64 常量折叠。
- 新增覆盖全部后端且可诊断退出的副作用 Float64 非零守卫，并以此为 PocketPy 原生除法保留 `ZeroDivisionError` 语义。
- 将托管自动化拆分为职责明确的持续验证、核心平台、语言基准、消毒器、发布元数据与发布工作流，不再使用笼统的 `ci.yml` 入口。

## 0.1.3

- 新增隔离的 PocketPy 2.1.8 构建、严格 Python 数值源码翻译、由 GC 管理生命周期的受守卫原生 callable，以及带校验和的 stock/UniJIT 调用基准。
- 新增隔离的 stock QuickJS 构建、严格数值函数到 Float64 SSA 的翻译、由 GC 管理代码生命周期的受守卫原生闭包，以及带校验和的 stock/UniJIT 调用基准。
- 新增显式执行上下文 ABI、无锁协作式中断、可诊断运行时退出，以及 AArch64、x86-64、RISC-V 64 上覆盖直线代码和 CFG 循环的副作用安全点。
- 新增可感知副作用的类型化运行时辅助调用、扁平 value-bits 参数、跨调用活跃值保护，以及所有目标架构上的原生 ABI 降级。
- 新增受守卫保护的 stock Lua 5.5 Float64 字节码编译，以及可复现的三引擎浮点调用基准。

## 0.1.2

- 新增类型化 Float64 SSA 常量与算术、IEEE-754 解释器 oracle、带优化的溢出处理，以及三个目标架构上的原生编码。
- 为全部原生后端新增 CFG 块内寄存器分配和可安全处理环路的并行寄存器复制，消除热循环中的栈往返。
- 将项目自有的前端、工具和文档目录统一改为单数形式：`frontend/`、`tool/` 与 `doc/`。
- 新增一个结构化 Lua 5.5 数值 `for` 循环的受守卫编译，覆盖 CFG 循环携带值、零次迭代语义与三引擎测试。
- 新增 AArch64、x86-64、RISC-V 64 上带范围校验的 CFG 原生降级，并以并行块参数复制正确处理循环回边。
- 新增使用显式块参数的 CFG SSA、支配关系验证，以及支持分支和循环并带执行预算的参考解释器。
- 新增 Lua 三引擎整数基准与锁定 ABI 的调用快路径；在本次调用边界工作负载上性能超过 stock Lua。
- 新增受守卫保护的 stock Lua 5.5 字节码到原生代码前端，首批支持直线整数函数，并配套差分与生命周期测试。
- 新增用于语言基准的隔离式 Lua 5.5 与 LuaJIT 参考构建。
